## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ

### 1. **Ошибка XP для польского языка** (types.ts:66)
**Серьезность:** ВЫСОКАЯ
**Текущее состояние:**
```typescript
export const TRACK_XP: Record<Track, number> = {
  polish: 20,  // ❌ НЕПРАВИЛЬНО
}
```

**Требование (из README):**
- `polish: 30 (1ч) / 15 (30мин)`

**Актуальный код в schedule.ts:137:**
```typescript
const xp = isLong ? 30 : 15  // ✅ Schedule правильно генерирует XP
```

**Проблема:**
- `TRACK_XP[track]` используется в `store.ts:59` при выполнении задачи
- Для польского это вернёт 20, но задача может иметь xp=30 или xp=15
- Будет несоответствие между XP в задаче и тем, что начисляется

**Решение:**
Либо удалить `polish` из `TRACK_XP` и всегда использовать `task.xp`, либо исправить логику в store при начислении XP для польского.

### 2. **Логика стрика неполная в completeTask** (store.ts:49-84)
**Серьезность:** СРЕДНЯЯ
**Текущее состояние:**

Стрик растет ТОЛЬКО если:
- Задача выполнена именно **сегодня** (line 61: `if (task.date === today)`)
- И это первая выполненная задача сегодня (line 62-65)

**Проблема:**
- Если пользователь не завершает никаких задач = стрик не растет
- Но в `processStreakOnOpen` (streak.ts) логика правильная = стрик растет на основе завершённых задач за прошлые дни
- **Несоответствие:** в одном месте стрик растёт по одному правилу, в другом — по другому

**Edge case:**
Если пользователь выполнит задачу завтра (заглянув в будущее), `completeTask` НЕ обновит стрик, но завтра при `processOnOpen` может обновиться.

**Решение:**
Полагаться на `processOnOpen` для всех вычислений стрика. Или синхронизировать логику между обеими функциями.

### 3. **Потенциальная потеря истории выполненных задач** (store.ts:118-135)
**Серьезность:** СРЕДНЯЯ
**Проблема в updateSchedule:**

```typescript
// Line 122-124
const keptTasks = s.tasks.filter(t =>
  !(t.isRecurring && t.date.startsWith(month)) ||
  t.completed || t.skipped
)
```

**Логика:** "Сохранить либо НЕ повторяющиеся, либо выполненные/пропущенные повторяющиеся"

**Edge case:**
- Если есть повторяющаяся задача, которая:
  - Не выполнена И
  - Не пропущена AND
  - Дата совпадает с рабочим днём этого месяца
- **Она удаляется и создаётся заново** ✓ Это правильно

**Но:** Нет явного уведомления пользователю о том, что задачи были пересгенерированы.

### 4. **Отсутствует обработка ошибок API** (route.ts:107-111)
**Серьезность:** СРЕДНЯЯ-НИЗКАЯ
```typescript
} catch (error) {
```

**Проблемы:**

- На клиенте в chat/page.tsx:137-138 вообще не обрабатывается ошибка
- Пользователь не видит информативного сообщения

**Решение:**
```typescript

if (!process.env.ANTHROPIC_API_KEY) {
  return NextResponse.json({ error: 'API ключ не настроен' }, { status: 500 })
}
```

### 5. **Timezone issues**
**Серьезность:** НИЗКАЯ (но важно помнить)

Весь код работает с **браузерным временем**:
- `format(new Date(), 'yyyy-MM-dd')` — локальная дата пользователя
- Нет UTC конверсии

**Пока работает**, но при масштабировании на облако нужно добавить синхронизацию.

## 📈 ПРОИЗВОДИТЕЛЬНОСТЬ

**localStorage usage:**
- Даны: ~1KB на 20 задач
- При 1000 задач: ~50KB — OK для localStorage (обычно лимит 5-10MB)

**API calls:**
- 1 запрос на сообщение в чат
- Нет throttling — если быстро кликнуть Send, может быть race condition
- **Решение:** добавить `disabled` кнопку во время загрузки ✓ Уже есть (`loading` state)
